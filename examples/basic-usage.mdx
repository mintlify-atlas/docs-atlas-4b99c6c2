## Getting Started

The most basic use case is finding the shortest path from Arad to Bucharest using the pre-configured Romania map and heuristics.

## Running the Script

The simplest way to use the pathfinder is to run the provided script directly:

```bash
python astar_shortest_path_romania.py
```

This will find the optimal path from Arad to Bucharest and display both the terminal output and a visualization window.

## Understanding the Script

The script `astar_shortest_path_romania.py` contains all the necessary components. Here's what happens when you run it:

```python
# The script calls a_star_search with the Romania map
optimal_path, total_cost = a_star_search(romania_map, 'Arad', 'Bucharest', heuristics)

# Prints the results
print("Ruta óptima de Fagaras a Bucarest:", " -> ".join(optimal_path))
print("Costo total de la ruta:", total_cost)

# Visualizes the graph with the optimal path highlighted
draw_graph(romania_map, optimal_path)
```

<Note>
The print statement in the source code says "Fagaras to Bucharest" but actually searches from Arad to Bucharest - this is a minor typo in the output message.
</Note>

## Expected Output

When you run this code, you'll see:

```
Optimal path from Arad to Bucharest: Arad -> Sibiu -> Rimnicu Vilcea -> Pitesti -> Bucharest
Total distance: 418 km
```

<Note>
The algorithm finds the path **Arad → Sibiu → Rimnicu Vilcea → Pitesti → Bucharest** with a total distance of **418 km**, which is the shortest possible route.
</Note>

## Understanding the Components

<Steps>

<Step title="Graph structure">
The `romania_map` dictionary contains all cities and their connections with distances.
</Step>

<Step title="Heuristic values">
The `heuristics` dictionary contains straight-line distance estimates from each city to Bucharest.
</Step>

<Step title="A* search function">
The `a_star_search()` function takes the graph, start city, goal city, and heuristics, returning a tuple of `(path, cost)`.
</Step>

<Step title="Visualization">
The `draw_graph()` function displays the Romania map with the optimal path highlighted in red.
</Step>

</Steps>

## Visualization

The `draw_graph()` function creates an interactive graph where:
- **Nodes** represent cities (shown in light blue)
- **Edges** represent roads with distances labeled
- **Red edges** highlight the optimal path found by A*

<Tip>
The graph layout is generated automatically using a spring layout algorithm, so node positions may vary between runs but the connections remain the same.
</Tip>

## What Happens Behind the Scenes

The A* algorithm uses:
1. **g(n)**: The actual cost from the start city to the current city
2. **h(n)**: The heuristic estimate from the current city to Bucharest
3. **f(n) = g(n) + h(n)**: The total estimated cost

The algorithm always explores the city with the lowest f(n) value first, ensuring an optimal solution when using admissible heuristics.
